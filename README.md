# apbd-cw1-git-s34264
· 1. Kiedy Git wykona fast-forward, a kiedy powstaje merge commit? \n
Git wykona fast-forward jeśli nie będzie po drodze żadnego konfliktu.
Merge commit jest efektem rozwiązania konfliktu.
· 2. Czym w praktyce różni się merge od rebase?
Merge łączy gałęzie wspólnym commitem a rebase przesuwa gałąź na czubek main.
· 3. W jaki sposób został rozwiązany konflikt w Twoim repozytorium?
Została wybrana jedna z możliwych nazw zmiennej.
